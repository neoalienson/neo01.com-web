我開發的 Hexo 插件

Created 2024-05-02   Updated 2025-10-25
  1. hexo-plugin-commentbox
  2. hexo-github-card-inline
  3. 為什麼開發這些插件？
  4. 安裝

使用 Hexo 建立靜態網站並不意味著要犧牲互動性。我開發了幾個插件來增強 Hexo 部落格的留言、GitHub 整合和其他功能。

💬 hexo-plugin-commentbox

將 commentbox.io 留言系統整合到你的 Hexo 部落格文章中。無需伺服器端基礎設施即可新增社群互動。

  • 預設在所有頁面啟用
  • 透過 front matter 精細控制
  • 簡單整合

🎴 hexo-github-card-inline

在文章中內嵌渲染 GitHub 使用者和儲存庫卡片。透過視覺化、資料豐富的展示來展示個人檔案和專案。

  • 帶統計資料的使用者檔案卡片
  • 帶指標的儲存庫卡片
  • 程式語言分布圖表

hexo-plugin-commentbox

hexo-plugin-commentbox 將 commentbox.io 留言系統整合到你的 Hexo 部落格文章中。

功能特性

  • 預設啟用：留言自動出現在所有頁面
  • 精細控制：透過 front matter 在特定文章停用留言
  • 簡單整合：只需最少的設定

停用留言

要在特定文章或頁面停用留言，在 front matter 中新增：

---
title: 你的文章標題
comments: false
---

安全注意事項

⚠️ 外部指令碼載入

此插件從 unpkg.com/commentbox.io 載入外部 JavaScript。請注意：

  • 外部指令碼可以存取你的頁面內容和使用者資料
  • 指令碼從 CDN 載入，沒有子資源完整性（SRI）驗證
  • 外部指令碼的變更不在你的控制範圍內
  • 考慮你網站的內容安全政策（CSP）要求

!!!

📝 免責聲明

此專案與 commentbox.io 沒有任何關聯、認可或連結。這是一個獨立插件，用於將 commentbox.io 服務整合到 Hexo 部落格中。

!!!

hexo-github-card-inline

hexo-github-card-inline 在文章中內嵌渲染 GitHub 使用者和儲存庫卡片。

使用者卡片

顯示包含統計資料和程式語言分布的完整 GitHub 使用者檔案：

{% raw %}{% githubCard user:neoalienson %}{% endraw %}

卡片顯示：

  • 📍 位置
  • 🔗 網站
  • 📁 儲存庫數量
  • ⭐ 總星標數
  • 📈 提交數
  • 👥 追蹤者
  • 🔄 拉取請求
  • ❗ 議題
  • 程式語言分布圖表

儲存庫卡片

展示特定儲存庫的關鍵指標：

{% raw %}{% githubCard user:neoalienson repo:pachinko %}{% endraw %}

卡片顯示：

  • 📖 儲存庫名稱和描述
  • ⭐ 星標數
  • 🍴 分支數
  • 主要程式語言

使用場景

  • 作品集頁面：展示你的 GitHub 個人檔案和專案
  • 專案文件：在技術文章中嵌入儲存庫卡片
  • 貢獻者認可：突顯團隊成員的 GitHub 個人檔案
  • 開源推廣：顯示專案統計資料以吸引貢獻者

為什麼開發這些插件？

像 Hexo 這樣的靜態網站產生器在效能和簡潔性方面表現出色，但它們缺乏開箱即用的動態功能。這些插件彌補了這一差距：

  • hexo-plugin-commentbox 無需伺服器端基礎設施即可新增社群互動
  • hexo-github-card-inline 提供 GitHub 內容的視覺化、資料豐富的展示

這兩個插件都遵循 Hexo 的標籤插件架構，使它們易於整合並在 markdown 內容中使用。

安裝

透過 npm 安裝：

npm install hexo-plugin-commentbox --save
npm install hexo-github-card-inline --save

在 _config.yml 中設定並開始在文章中使用它們。

這些插件是開源的，可在 GitHub 上取得。歡迎貢獻、提出議題和回饋。
