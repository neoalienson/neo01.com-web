我开发的 Hexo 插件

Created 2024-05-02   Updated 2025-10-25
  1. hexo-plugin-commentbox
  2. hexo-github-card-inline
  3. 为什么开发这些插件？
  4. 安装

使用 Hexo 构建静态网站并不意味着要牺牲交互性。我开发了几个插件来增强 Hexo 博客的评论、GitHub 集成和其他功能。

💬 hexo-plugin-commentbox

将 commentbox.io 评论系统集成到你的 Hexo 博客文章中。无需服务器端基础设施即可添加社区互动。

  • 默认在所有页面启用
  • 通过 front matter 精细控制
  • 简单集成

🎴 hexo-github-card-inline

在文章中内联渲染 GitHub 用户和仓库卡片。通过可视化、数据丰富的展示来展示个人资料和项目。

  • 带统计数据的用户资料卡片
  • 带指标的仓库卡片
  • 编程语言分布图表

hexo-plugin-commentbox

hexo-plugin-commentbox 将 commentbox.io 评论系统集成到你的 Hexo 博客文章中。

功能特性

  • 默认启用：评论自动出现在所有页面
  • 精细控制：通过 front matter 在特定文章禁用评论
  • 简单集成：只需最少的配置

禁用评论

要在特定文章或页面禁用评论，在 front matter 中添加：

---
title: 你的文章标题
comments: false
---

安全注意事项

⚠️ 外部脚本加载

此插件从 unpkg.com/commentbox.io 加载外部 JavaScript。请注意：

  • 外部脚本可以访问你的页面内容和用户数据
  • 脚本从 CDN 加载，没有子资源完整性（SRI）验证
  • 外部脚本的变更不在你的控制范围内
  • 考虑你网站的内容安全策略（CSP）要求

!!!

📝 免责声明

此项目与 commentbox.io 没有任何关联、认可或连接。这是一个独立插件，用于将 commentbox.io 服务集成到 Hexo 博客中。

!!!

hexo-github-card-inline

hexo-github-card-inline 在文章中内联渲染 GitHub 用户和仓库卡片。

用户卡片

显示包含统计数据和编程语言分布的完整 GitHub 用户资料：

{% raw %}{% githubCard user:neoalienson %}{% endraw %}

卡片显示：

  • 📍 位置
  • 🔗 网站
  • 📁 仓库数量
  • ⭐ 总星标数
  • 📈 提交数
  • 👥 关注者
  • 🔄 拉取请求
  • ❗ 议题
  • 编程语言分布图表

仓库卡片

展示特定仓库的关键指标：

{% raw %}{% githubCard user:neoalienson repo:pachinko %}{% endraw %}

卡片显示：

  • 📖 仓库名称和描述
  • ⭐ 星标数
  • 🍴 分支数
  • 主要编程语言

使用场景

  • 作品集页面：展示你的 GitHub 个人资料和项目
  • 项目文档：在技术文章中嵌入仓库卡片
  • 贡献者认可：突出显示团队成员的 GitHub 个人资料
  • 开源推广：显示项目统计数据以吸引贡献者

为什么开发这些插件？

像 Hexo 这样的静态网站生成器在性能和简洁性方面表现出色，但它们缺乏开箱即用的动态功能。这些插件弥补了这一差距：

  • hexo-plugin-commentbox 无需服务器端基础设施即可添加社区互动
  • hexo-github-card-inline 提供 GitHub 内容的可视化、数据丰富的展示

这两个插件都遵循 Hexo 的标签插件架构，使它们易于集成并在 markdown 内容中使用。

安装

通过 npm 安装：

npm install hexo-plugin-commentbox --save
npm install hexo-github-card-inline --save

在 _config.yml 中配置并开始在文章中使用它们。

这些插件是开源的，可在 GitHub 上获取。欢迎贡献、提出议题和反馈。
